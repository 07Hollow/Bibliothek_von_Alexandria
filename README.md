# Knowledge Base

This repository contains a comprehensive knowledge (in German) base on
- Biology
- Chemistry
- Electrical engineering
- Computer Science
- Math
- Physics
- Economy

Ceated with Obsidian, a powerful knowledge management tool, and exported using the Perlite plugin, this project aims to provide an easily accessible and navigable resource for learning and reference. The static site is hosted on Netlify for seamless access and hosting.

## Features

- **Extensive Content**: Covers a wide range of topics in Physics, Computer Science, and more.
- **Obsidian Integration**: Utilizes Obsidian for note-taking and organization.
- **Static Site Generation**: Uses the Perlite plugin to generate static HTML files from Obsidian notes.
- **Netlify Deployment**: Deployed on Netlify for seamless access and hosting.

## Usage

### Browsing the Content

The knowledge base is accessible as a static website. You can browse through the various topics and notes for detailed information and learning resources.

### Updating Content

1. **Clone the Repository**:
    ```bash
    git clone https://github.com/07Hollow/Bibliothek_von_Alexandria
    cd Bibliothek_von_Alexandria
    ```

2. **Make Changes in Obsidian**:
    - Open your Obsidian vault containing the notes.
    - Make your changes or add new notes.

3. **Export with Perlite**:
    - Use the Perlite plugin in Obsidian to generate the static HTML files.

4. **Commit and Push Changes**:
    ```bash
    git add .
    git commit -m "Update notes"
    git push origin main
    ```

## Deployment

The static site generated by Perlite is hosted on Netlify. Any updates pushed to this repository are automatically deployed to keep the site up-to-date.

### Initial Setup on Netlify

1. **Create a New Site**:
    - Log in to Netlify and click on "New site from Git".
    - Connect your GitHub repository.

2. **Configure Build Settings**:
    - **Build command**: None (as the static files are pre-generated).
    - **Publish directory**: The directory where your Perlite-generated files are located (e.g., `dist` or `public`).

3. **Deploy Site**:
    - Click on "Deploy site" to publish your knowledge base.
